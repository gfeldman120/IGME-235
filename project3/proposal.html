Process

The first thing I did was make the GameObject class and get things appearing in an HTML grid. I needed to make sure that things were displaying exactly to where I needed them.
Following that, I got player input working through adding events that register on the "document." This ensures that the player doesn't have to click into the grid to move.
Then, I focused on collision. This took the most time due to problem solving: I wanted the player to be allowed to "slide" on surfaces if they were moving diagonally but could only move in 1 direction.
Once this was done, I went about implementing the score system using simple position checking with the goal object and the local storage system for the high score.
I recieved feedback that the player movement felt too "jagged" so I played around with CSS transitions to create a trail behind the player.
Finally, I went about styling the page so it was (at least somewhat) appealing to look at. It was suggested that I use gradients for certain elements - I'm glad with how it came out (though I'm no graphic designer)!

Meeting requirements

  1. Functional: Uses Javascript + browser DOM, only needs button click + WASD, hasn't been done in class, no errors (that I've found)
  2. Design & Interaction: Simple colors that don't hurt the eyes, simple labels are easy to read, basic instruction provided, works on 1024 x 768 at minimum
  3. Media: No image files present - therefore no need to optimize, SFX used to supplement visual information
  4. HTML/CSS: HTML and CSS validated before submission, all CSS is external, HTML goes in order of page elements from top to bottom, left to right (in this order)
  5. Code Conventions: GameObject class is used, only "let" and "const" are used, functions used to avoid repetition, GameObject class is its own file, variable and function names are lowercase, all functions + some extra sections commented, no console.log

Proposal (Documentation at top)

Due to the limited time frame for this project, I'm keeping my game simple. The player will be represented by a green cube. Their objective is to reach a blue cube within the given time frame. Randomly placed red cubes will block the player's path.
There is also a timer outside of the game view. If the timer runs out, the player must restart. The timer is extended by hitting the blue cube. When the blue cube is hit, it will move to a new location.
The red cubes will also swap places when the blue cube is hit or when the game starts. The player can wrap around the screen if need be.
